# 【緊急】React2Shell（CVE-2025-55182）完全解説 ── React/Next.jsに最大深刻度のRCE脆弱性、中国系APTが即日攻撃開始

2025年12月3日、ReactおよびNext.jsに**CVSSスコア10.0（最大深刻度）**のリモートコード実行（RCE）脆弱性が公開されました。「**React2Shell**」と名付けられたこの脆弱性は、認証なしで任意のコード実行が可能であり、公開からわずか数時間で中国系APT（高度標的型攻撃）グループによる攻撃が確認されています。

本記事では、フロントエンドエンジニア・バックエンドエンジニアが知るべき技術的詳細から、緊急対応手順までを詳しく解説します。

---

## TL;DR（3行まとめ）

1. **React 19.x系のServer Components**に認証不要のRCE脆弱性（CVSSスコア10.0）
2. **Next.js 15.x/16.x（App Router使用時）**が特に影響大、39%のクラウド環境が脆弱
3. **即座にパッチ適用が必要**：公開数時間で中国系脅威グループによる攻撃を確認

## 技術的詳細

### 脆弱性のメカニズム

React2Shellは、**React Server Components（RSC）の「Flight」プロトコル**におけるデシリアライズ処理の欠陥を悪用します。

```
[クライアント] → HTTP Request → [サーバー]
                                    ↓
                            RSC Flight Protocol
                                    ↓
                            デシリアライズ処理 ← ここに脆弱性
                                    ↓
                            任意コード実行（RCE）
```

**具体的な攻撃フロー：**

1. 攻撃者が悪意のあるHTTPリクエストを作成
2. React Server Functionエンドポイントに送信
3. サーバー側でFlightプロトコルがペイロードをデシリアライズ
4. 不正なデータ構造により、サーバー側の実行ロジックが汚染
5. 任意のJavaScriptコードがサーバー上で実行される

### なぜ深刻なのか

セキュリティ企業Wizによると、この脆弱性には以下の特徴があります：

- **認証不要**：ログインなしで攻撃可能
- **リモート実行**：HTTPリクエストのみで攻撃可能
- **デフォルト設定で脆弱**：特別な設定なしで標準デプロイが影響を受ける
- **高い成功率**：実験では「ほぼ100%の成功率」で攻撃が成功
- **Server Functionsを使っていなくても脆弱**：RSCをサポートしているだけで影響

### 重要な注意点

発見者のLachlan Davidson氏は、偽のPoC（概念実証）が出回っていることを警告しています：

> 「`vm#runInThisContext`、`child_process#exec`、`fs#writeFile`などを開発者が明示的に公開している前提のPoCは無効です。真の脆弱性にはそのような制約はありません。」

これらの偽PoCに基づいて「脆弱でない」と判断することは危険です。

## 影響範囲

### 統計データ

Wizのクラウドセキュリティデータによると：

- **39%のクラウド環境**が脆弱なReact/Next.jsインスタンスを保有
- **69%の環境**にNext.jsが存在
- Next.jsを持つ環境の**61%**が公開アプリケーションを運用
- 全クラウド環境の**44%**がNext.jsを公開している

Palo Alto Networks Unit 42によると：

- **968,000台以上のサーバー**がReact/Next.jsを実行
- これらが潜在的な攻撃対象となる

### 攻撃の実態

**AWSの脅威インテリジェンスチームは、公開から数時間以内に以下の中国系APTグループによる攻撃を確認：**

- **Earth Lamia**：中国系の脅威アクター
- **Jackpot Panda**：中国系の脅威アクター

AWSのMadPotハニーポットシステムで攻撃試行を検知しており、匿名化インフラの共有により正確な帰属は困難ながらも、中国国家関連の活動であることが示唆されています。

## 緊急対応手順

### Step 1: 影響確認

以下の条件に該当する場合、脆弱性の影響を受けます：

```bash
# package.jsonを確認
cat package.json | grep -E "(next|react-server-dom)"
```

**影響を受けない条件：**
- React 18以下のみを使用
- クライアントサイドレンダリング（CSR）のみ
- React Server Componentsを使用していない
- Pages Routerのみ使用（Next.js）

### Step 2: 即座にアップデート

#### Next.js

```bash
# 使用中のバージョンに応じて適切なパッチを適用
npm install next@15.0.5   # 15.0.x の場合
npm install next@15.1.9   # 15.1.x の場合
npm install next@15.2.6   # 15.2.x の場合
npm install next@15.3.6   # 15.3.x の場合
npm install next@15.4.8   # 15.4.x の場合
npm install next@15.5.7   # 15.5.x の場合
npm install next@16.0.7   # 16.0.x の場合

# canary版（14.3.0-canary.77以降）を使用している場合
npm install next@14       # 安定版14.xにダウングレード
```

#### React Server DOM パッケージ

```bash
npm install react@latest react-dom@latest
npm install react-server-dom-webpack@latest
# または
npm install react-server-dom-parcel@latest
# または
npm install react-server-dom-turbopack@latest
```

#### その他のフレームワーク

```bash
# Waku
npm install waku@latest react@latest react-dom@latest react-server-dom-webpack@latest

# React Router（RSC使用時）
npm install react@latest react-dom@latest @vitejs/plugin-rsc@latest

# Redwood SDK
npm install rwsdk@latest react@latest react-dom@latest react-server-dom-webpack@latest
```

### Step 3: WAF保護の確認

主要クラウドプロバイダーは既にWAFルールをデプロイしています：

| プロバイダー | 対応状況 |
|-------------|---------|
| **Cloudflare** | 自動保護（プロキシ経由のトラフィック） |
| **AWS** | AWSManagedRulesKnownBadInputsRuleSet（v1.24以上）で対応 |
| **Google Cloud** | Cloud Armor WAFルール追加 |
| **Akamai** | WAFルール追加 |
| **Fastly** | WAFルール追加 |

**注意**：WAFは一時的な緩和策であり、根本的な解決にはパッチ適用が必須です。

### Step 4: ログの確認

攻撃の兆候がないか、アクセスログを確認してください：

- 異常なPOSTリクエスト
- Server Functionエンドポイントへの不審なアクセス
- 500 Internal Server Errorの急増（攻撃試行のサインである可能性）

## Log4Shellとの比較

Bugcrowdの分析によると：

> 「この脆弱性は重大ですが、Log4jほどのスケールではありません。初期のクラウドテレメトリでは、監視対象環境の約3分の1で露出が確認されています。深刻で、広範囲で、緊急ですが、インターネット全体に影響するわけではありません。」

ただし、以下の点で注意が必要です：

- **Reactの普及率は82%**（State of JavaScript 2024調査）
- **Next.jsは最も人気のあるReactフレームワーク**
- **App Routerの採用が進んでいる**ため、影響は今後拡大する可能性

## 開発者への推奨事項

### 短期（今すぐ）

1. **パッチを即座に適用**
2. **CI/CDパイプラインで依存関係を更新**
3. **WAF保護が有効か確認**
4. **アクセスログを監視**

### 中期（1週間以内）

1. **依存関係スキャンツールの設定確認**
   - Next.jsはReactをvendored形式でバンドルするため、通常の依存関係スキャンでは検出されない可能性
2. **セキュリティ監査の実施**
3. **インシデント対応計画の見直し**

### 長期

1. **RSCを使用するかどうかの再評価**
2. **サーバーサイド実行のセキュリティベストプラクティス見直し**
3. **Zero-day対応プロセスの整備**

## まとめ

React2Shell（CVE-2025-55182）は、React/Next.jsエコシステムにおける最も深刻な脆弱性の一つです。

**今すぐ対応すべき理由：**

- CVSSスコア10.0（最大深刻度）
- 認証不要でリモートコード実行が可能
- 中国系APTグループが既に攻撃を開始
- デフォルト設定で脆弱
- 39%のクラウド環境が影響を受ける

**パッチ適用を最優先で実施してください。**
